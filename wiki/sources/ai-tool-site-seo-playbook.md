# AI Tool Site SEO Playbook

Source summary for `raw/sources/seo.md`, an unfinished Chinese SEO playbook focused on AI tool websites.

## Summary

The source argues that AI tool site SEO should be treated as a practical growth system rather than a collection of ranking tricks. The operating model is to define the search user, infer page types from search intent and SERP patterns, build technical foundations early, create pages with real information gain, and grow off-site trust through relevant backlinks.

Its central framing is that a website should be managed like a book and a content product: the site is the book, pages are chapters, internal links express chapter importance, and backlinks are external trust votes. The practical goal is for a search user to land on the page, complete the task, and stop searching elsewhere.

## Practical SEO System

The source repeatedly decomposes SEO into three execution systems:

1. **Structure and technical system** — crawlability, indexability, URL structure, page templates, structured data, Core Web Vitals, and on-page hygiene.
2. **Content and product system** — search intent, page type matching, task completion, information gain, and conversion path.
3. **Trust system** — backlinks, citations, AI directories, review sites, referrals, and external authority signals.

A concise formula appears in the source:

> SEO 排名 = 页面内容 + 页面基建 + 外链

## Key Claims

- Modern SEO is still about satisfying search demand, but Google and AI-assisted search make the work more system-oriented.
- AI tool pages should be designed from search intent backward: keyword → user task → page type → content modules → CTA.
- `Last-Click` is a useful goal: the page should solve the searcher's task well enough that they do not return to the results page.
- New AI tool sites should not begin with broad head terms. They should prioritize long-tail or blue-ocean keywords, then expand toward mid-size and head terms.
- Keyword prioritization should weigh conversion intent, not just search volume. In a zero-click environment, commercial and transactional intent can be more valuable than purely informational content.
- SEO infrastructure should be planned from the beginning if SEO will become a meaningful channel; late restructuring of routes, templates, schema, and indexation is expensive.
- Backlinks are a trust system, but unnatural or overly concentrated growth can be risky.
- Programmatic SEO only works when templates scale real information gain. Thin AI-generated pages are treated as low-quality at scale.

## Execution Workflow Extracted from the Source

1. Define the [[concepts/seo-icp|SEO ICP]].
2. Map keyword clusters to user tasks and intent.
3. Inspect SERP page types before choosing a page format.
4. Build the [[concepts/ai-tool-site-seo-foundation|SEO foundation]] before scaling pages.
5. Create pages that satisfy one intent with clear modules: conclusion, steps, comparison, FAQ, examples, screenshots, data, or demos.
6. Use internal links to route importance from hubs to clusters and leaf pages.
7. Build relevant and diverse backlinks, starting with practical AI directory and review-site submissions where appropriate.
8. Track non-brand organic clicks, effective index rate, top-page coverage, referring domains, and conversions on money pages.
9. For pSEO, validate a small batch of useful pages before scaling templates.

## Limitations and Uncertainty

- The source is an unfinished playbook, with some sections marked as still in progress.
- The file appears to contain OCR or extraction noise, including malformed tables, duplicated lines, and unrelated fragments.
- Many tactical claims are experience-based SEO advice rather than independently verified facts.
- Backlink, directory submission, and “parasite SEO” recommendations should be treated as strategy claims, not neutral best practices.
- Some examples refer to third-party tools and metrics such as Ahrefs, SEMRush, Similarweb, GSC, and GA4 without fully specifying measurement methodology.
