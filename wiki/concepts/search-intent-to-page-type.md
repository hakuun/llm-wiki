# Search Intent to Page Type

Search intent to page type is the practice of mapping a keyword to the user task and the page form Google appears to prefer for that task. The source frames search intent as both a user requirement and a page-shape requirement.

## Core Idea

SEO should not be planned as one keyword equals one page. A better rule is one dominant search intent equals one page. If the same keyword has multiple serious intents, those intents may require different pages.

## Common Intent Types

| Intent | User wants | Common AI tool site page |
| --- | --- | --- |
| Informational | Learn or understand | Tutorial, guide, how-to article |
| Commercial | Compare and decide | Best-list, review, comparison, alternatives page |
| Transactional | Use or buy | Tool landing page, demo, signup, pricing, download |
| Navigational | Reach a known brand or page | Brand, login, docs, API entry page |

## SERP Check

Before choosing a page type, inspect the top results:

- What page forms dominate: tools, listicles, tutorials, UGC, docs, or landing pages?
- What content format is common: table, list, review, step-by-step guide, FAQ, demo, or template?
- What angle is repeated: free, latest, batch, API, privacy, open-source, enterprise, or specific model support?

## Execution Fields

Add these fields to keyword planning:

- Search intent.
- Dominant SERP page type.
- Required content modules.
- Page CTA.
- Evidence required for information gain.
